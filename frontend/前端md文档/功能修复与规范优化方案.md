# 前端功能修复与组件规范优化方案

## 1. 核心修复目标
解决搜索条件不生效、状态字段文案错误、新增业务状态以及详情页显示异常的问题。同时，建立“组件与页面同级”的开发规范。

## 2. 详细修复任务

### 2.1 搜索逻辑对齐 (SearchFilter.jsx)
*   **字段对齐**：确保 `Form.Item` 的 `name` 属性与后端 API 参数 (`keyword`, `std_type`, `status`) 100% 一致。
*   **Select 事件**：显式设置 `Select` 的 `options` 值，确保点击即触发 `Form` 状态更新。

### 2.2 业务字段与状态扩充
*   **全局更名**：
    *   “有效性” $\rightarrow$ **“执行状态”**。
*   **新增状态逻辑**：
    *   状态码 `2` 定义为 **“即将实施”**。
    *   视觉表现：使用极净蓝标签 (`bg-blue-50`, `text-blue-600`, `ring-blue-100/50`)。

### 2.3 详情页 (DetailPage.jsx) 功能还原
*   **逻辑迁移**：参考 `SearchPage_old.jsx` 中的 AJAX 请求方式，在 `DetailPage` 中使用 `useParams` 获取 ID，并实现异步数据加载。
*   **视觉重构**：将详情页也升级为“极智毛玻璃”风格，保持全站视觉统一。

### 2.4 目录规范化
*   所有业务组件路径强制要求：
    *   `src/pages/StandardSearch/components/...`
    *   `src/pages/DetailPage/components/...`

## 3. 技术实施路径
1.  **Step 1**: 更新 `SearchFilter.jsx` 修复搜索点击无效。
2.  **Step 2**: 更新 `StandardTable.jsx` 修改表头文字并增加“即将实施”状态渲染。
3.  **Step 3**: 全面重写 `src/pages/DetailPage.jsx`。
4.  **Step 4**: 调整全局 CSS 适配新增状态色。
