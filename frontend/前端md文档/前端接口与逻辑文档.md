# 标准信息平台 - 前端网络请求与 API 逻辑文档

该文档旨在说明前端 React 架构中是如何封装 Axios、管理全局 Token 以及调用后端 API 的。

## 1. 核心封装 (`src/api/axios.js`)

前端采用了单例模式的 Axios 封装，拦截了所有出入的请求：
- **请求拦截器 (Request Interceptor)**：从 `localStorage` 中获取 `token`，并在向后端发起请求前，自动注入 `Authorization: Bearer <token>` 到请求头中。
- **响应拦截器 (Response Interceptor)**：
  - **自动脱壳**：拦截器会自动读取后端的 `{ code: 0, data: {...} }` 结构，如果 `code === 0`，则直接将 `data` 抛给业务层（业务代码无需再写 `res.data.data`）。
  - **错误处理**：如果 `code !== 0`，会自动转换为 Promise 错误抛出，组件内的 `catch` 可以直接捕获并 `message.error()`。
  - **权限失效兜底**：如果后端返回 HTTP `401 Unauthorized`（如 Token 过期），拦截器会自动清除本地的过期 Token 并通过 `window.location.href = '/login'` 强制登出。
  - **文件流放行**：针对 `/download` 接口设置了白名单（判断 `responseType === 'blob'`），不会被脱壳逻辑影响。

## 2. 接口定义 (`src/api/standards.js`)

所有的网络请求按业务进行了函数拆分，返回的均是 `Promise` 对象：

### 2.1 认证模块
- `login({ username, password })`: 用户登录，成功后获取 token。

### 2.2 检索与基础数据
- `searchStandards(params)`: ES/MySQL 全文检索，传入 `{ keyword, std_type, status, page, size }`。
- `getStandardDetail(stdId)`: 传入 `stdId` (注意：该函数内部会自动 `encodeURIComponent()` 防止 `/` 破坏路由)。
- `downloadStandardFile(stdId)`: 发送文件下载请求（强制声明了 `responseType: 'blob'`）。

### 2.3 异步大模型任务
- `parseReferences(stdId)`: 发起标准规范引用抽取任务。
- `evaluateCompliance(stdId, businessDesc)`: 结合业务描述，触发大模型合规评价任务。
- `checkTaskStatus(taskId)`: 轮询查询上述两个任务的执行结果。

## 3. 轮询引擎 (`src/hooks/useTaskPolling.js`)

由于调用 Dify 大模型可能需要几十秒的时间，前端不可采用一直等待（阻塞模式），因此我们自研了心跳轮询 Hook。

**工作原理**：
1. 页面（如 `DetailPage.jsx`）触发大模型请求，并拿到后端返回的 `task_id`。
2. 页面调用 `startPolling(task_id)`。
3. `useTaskPolling` 内部启动一个 `setInterval` 定时器，**每隔 3 秒** 向后端发送一次 `checkTaskStatus`。
4. 如果接口返回 `status === 'SUCCESS'` 或 `'FAILURE'`，则主动清除定时器（`clearInterval`）终止轮询，并将大模型输出赋值给 `taskResult`，最后在页面上渲染出 AI 报告。
5. 当用户离开详情页时，`useEffect` 的销毁函数会自动清理后台定时器，**彻底杜绝了前端内存泄漏现象**。
