# 增加注册功能 - 前端架构设计与实施文档

## 1. 核心目标
重构并提升现有登录与注册模块的视觉审美（采用更现代的 Tailwind CSS 卡片设计），并且严格确保组件职责单一（UI 视图层绝对不包含底层 HTTP 请求代码）。

## 2. 实施路径

### 2.1 独立 API 封装层 (`src/api/auth.js`)
- **规范**：禁止在 UI 组件内部直接写 `axios.post`。
- **动作**：新建 `auth.js` 统一收拢认证类的请求，在此文件中封装导出 `registerApi(data)` 方法，内部负责组装 Payload 并发往 `/api/v1/auth/register`。

### 2.2 新建专属注册组件 (`src/pages/Register.jsx`)
- **文件独立**：专门承载“管理员分配新账号”的表单交互，与登录页面完全剥离。
- **纯粹的视图层**：
  - 引入 React `useState` 集中管理表单录入状态（username, password, confirmPassword, role）。
  - 仅做前置基础校验（如两次密码是否一致）。
  - 提交按钮的 `onClick` 事件只负责调用 `registerApi` 并处理 Ant Design 的 `message` 反馈。
- **样式方案**：使用 Tailwind CSS 工具类设计居中、圆角、带有悬浮反馈 (`hover:shadow-lg`) 的现代化卡片 UI，避免臃肿的行内 style。

### 2.3 登录页视觉重塑 (`src/pages/Login.jsx`)
- **痛点**：目前登录页输入框贯穿满屏，排版简陋。
- **改造方案**：
  - 背景：加入淡雅的网格背景或软渐变底色 (`bg-gray-50`)。
  - 容器：将表单约束在拥有高斯模糊（Glassmorphism）和深层阴影的 `max-w-md` 卡片盒子里。
  - 交互：增强 Input 的 `focus:ring-indigo-500` 发光边框与按钮的平滑过渡动画，呈现企业级高端质感。

### 2.4 路由集成 (`src/App.jsx`)
- **入口隔离**：因为只有超级管理员才能注册人员，`/register` 路由将被放置在 `AdminLayout`（已登录验证容器）下。超级管理员可以在系统侧边栏点击进入该页面完成人员注册分配。
