# 标准管理系统：标准谱系全链路可视化实现指南 (V2.0 完整版)

本手册旨在提供一套完整的技术方案，用于将非结构化的标准替代关系转化为可交互、高颜值的树状拓扑图。

---

## 第一阶段：数据库底层设计 (数据模型)

要复现本方案，数据库必须具备以下三个核心表：

1.  **`std_replace` (替代关系表 - 连线来源)**
    *   `base_id`: 当前标准 ID
    *   `replace_id`: 被替代标准的 ID (若库中无此标准，则存为 NULL)
    *   `replace_std_name`: 被替代标准的标准号字符串

2.  **`std_pedigree` (谱系归属表 - 家族索引)**
    *   `base_id`: 标准 ID
    *   `ped_id`: 家族唯一 ID (同一家族的标准共享同一个 ped_id)

3.  **`std_ped_chain` (谱系缓存表 - 渲染源)**
    *   `ped_id`: 家族唯一 ID
    *   `ped_chain`: **核心字段 (TEXT)**，存储计算好的 JSON 数据。

---

## 第二阶段：后端递归算法 (DFS 实现)

当有新标准 A 入库或执行数据迁徙时，调用以下 Python 逻辑生成 JSON：

### 核心 DFS 算法示例：
```python
def generate_pedigree_json(root_std_id, cursor):
    nodes = set(); edges = []; visited = set()
    
    def dfs(curr_id):
        if curr_id in visited: return
        visited.add(curr_id)
        # 获取标准号
        name = get_std_name_by_id(curr_id)
        nodes.add(name)
        
        # 查找该标准替代了谁
        cursor.execute("SELECT replace_id, replace_std_name FROM std_replace WHERE base_id = %s", (curr_id,))
        for r_id, r_name in cursor.fetchall():
            target_name = get_std_name_by_id(r_id) if r_id else r_name
            nodes.add(target_name)
            edges.append({"source": name, "target": target_name})
            if r_id: dfs(r_id) # 递归向下钻取

    dfs(root_std_id)
    return json.dumps({"nodes": list(nodes), "edges": edges})
```

## 第三阶段：前端数据协议 (JSON 格式)

后端写回 `std_ped_chain.ped_chain` 的数据必须严格遵守以下格式，否则前端无法解析：
```json
{
  "nodes": ["A", "B", "C", "D"],
  "edges": [
    {"source": "A", "target": "B"},
    {"source": "A", "target": "C"},
    {"source": "B", "target": "D"}
  ]
}
```

## 第四阶段：前端可视化与布局算法 (ECharts)

### 1. 核心布局算法 (解决“混乱”问题的关键)
前端拿到 JSON 后，不能直接展示，需要通过 JS 计算每个节点的坐标（x, y），使其呈现树状。

```javascript
// 计算层级 (Level) 算法
const levels = {};
const calculateLevels = (nodes, edges) => {
    const getLvl = (name) => {
        if (levels[name] !== undefined) return levels[name];
        const incoming = edges.filter(e => e.target === name);
        if (incoming.length === 0) return levels[name] = 0; // 根节点
        return levels[name] = Math.max(...incoming.map(e => getLvl(e.source))) + 1;
    };
    nodes.forEach(getLvl);
};

// 2. 坐标分配算法 (含重心启发式排序，减少线条交叉)
const generateCoords = (nodes, edges, orient = 'vertical') => {
    // 步骤 A: 逐层计算重心 (Barycenter) 并排序
    for (let l = 0; l <= maxLevel; l++) {
        const nodesInLevel = levelGroups[l];
        if (l > 0) {
            nodesInLevel.sort((a, b) => {
                const getAvgParentX = (n) => {
                    const parents = edges.filter(e => e.target === n).map(e => e.source);
                    if (parents.length === 0) return 0;
                    // 重心值 = 所有父节点在上一层位置的平均值
                    return parents.reduce((sum, p) => sum + nodePosX[p], 0) / parents.length;
                };
                return getAvgParentX(a) - getAvgParentX(b);
            });
        }
        // 步骤 B: 根据排序后的顺序分配 X 坐标
        nodesInLevel.forEach((name, idx) => {
            const xGap = Math.max(220, nodesInLevel.length * 30); // 动态间距
            nodePosX[name] = (idx - (nodesInLevel.length - 1) / 2) * xGap;
        });
    }
};
    return nodes.map(n => ({
        name: n.name,
        x: (index_in_level - center_offset) * spacing, // 平摊分布
        y: n.level * row_height, // 层级垂直分布
    }));
};
```

## 六、 进阶：如何处理“多父节点”？
本方案采用的 **Graph + Manual Layout** 模式完美支持 DAG（有向无环图）。如果一个标准被两个新标准共同替代，布局算法会自动将其放置在较低的层级，并同时接受来自上方的两条连线，逻辑完全闭环。
