# 项目启动指令指南

> 完整架构与开发说明见 **[docs/开发者指南.md](docs/开发者指南.md)** 与根目录 [README.md](README.md)。

本文档记录了 **标准检索页面项目 (Standard Search Page)** 的本地启动流程。

## 1. 环境准备 (前提条件)

在启动项目之前，请确保以下基础服务已安装并运行：

- **MySQL**: 数据库名称为 `mydate`，端口 `3306`。
- **Redis**: 用于 Celery 任务队列，默认端口 `6379`。
- **Node.js**: 建议版本 18+。
- **Python**: 建议版本 3.10+。

---

## 2. 启动后端服务 (Django)

后端默认运行在 **8001** 端口（为避免与本地其他项目的 8000 端口冲突）。

```powershell
# 1. 进入后端目录
cd backend

# 2. 激活虚拟环境 (Windows)
.\venv\Scripts\activate

# 3. 启动开发服务器
python manage.py runserver 8001
```

- **API 访问地址**: `http://127.0.0.1:8001/api/v1`
- **后台管理地址**: `http://127.0.0.1:8001/admin`

---

## 3. 启动前端服务 (Vite + React)

前端会自动连接到后端的 8001 端口。

```powershell
# 1. 进入前端目录
cd frontend

# 2. 启动开发服务器
npm run dev
```

- **访问地址**: [http://localhost:5173/](http://localhost:5173/)

---

## 4. 启动异步任务 (Celery) - 仅 AI 分析功能需要

如果您需要使用 **AI 规范性引用解析** 或 **合规性评价** 功能，必须启动 Celery Worker。

```powershell
# 1. 确保 Redis 已启动
# 2. 进入后端目录并激活虚拟环境
cd backend
.\venv\Scripts\activate

# 3. 启动 Celery Worker
celery -A config worker -l info
```

## 6. 服务器部署与登录变慢排查

本地快、服务器上登录「转圈很久」，多数是**登录后的自动查询**或**环境配置**，不是账号校验本身。

### 6.1 登录后实际发生了什么

1. `POST /api/v1/auth/login` 校验账号（通常几百毫秒内）
2. 跳转到 **标准检索** 页
3. 按设计会 **自动用默认条件查一次**（空关键词全库第一页）

第 3 步在数据量大时，MySQL 做 `COUNT(*)` 会很慢，容易误以为「登录慢」。

### 6.2 必查配置

| 项 | 说明 |
|----|------|
| `frontend/.env.production` | `VITE_API_BASE` 必须指向服务器上的后端，例如 `https://你的域名/api/v1`，不能仍是 `127.0.0.1` |
| `backend` 数据库 | `settings.py` 或环境变量里 `HOST` 不要用本机 `127.0.0.1`，应填 MySQL 实际地址 |
| Redis | `CACHES` / Celery 的 Redis 地址在服务器上要可达，连不上会导致接口卡顿 |
| Elasticsearch | 未部署 ES 时务必设 `USE_ELASTICSEARCH=false`（项目默认已是 false），否则可能每次检索先等 ES 超时 |

### 6.3 浏览器里如何区分

打开开发者工具 → **Network**：

- 只有 `auth/login` 很慢 → 查数据库连接、后端是否同机、防火墙
- `login` 很快，但 `standards/search` 很慢 → 属正常数据查询压力，可依赖 Redis 缓存或缩小默认检索范围

### 6.4 已做的体验优化

- 登录成功后 **先跳转** 再提示，减少「卡在登录页」的感觉
- 默认全量检索 **先返回列表、再后台补总数**，避免登录后长时间白屏

---

## 常见问题处理
1. **端口冲突**: 如果 8001 端口被占用，请在启动后端时更换端口（如 `python manage.py runserver 8002`），并同步修改 `frontend` 构建环境变量 `VITE_API_BASE`。
2. **数据库错误**: 如果提示 `OperationalError`，请检查 `backend\config\settings.py` 中的数据库配置是否正确。
