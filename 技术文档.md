3. 目录架构与职责划分 (关键指令)
请 AI 在生成代码时，严格遵循以下文件职责划分：

【后端 Django 模块化要求】
models.py: 仅定义数据库表结构（参考已有的 ER 图）。

serializers.py: 仅负责数据序列化与 Pydantic 风格的数据校验。

crud.py: 封装最底层的数据库 get/create/filter 操作，不写逻辑。

services.py: 核心！ 所有的逻辑（计算、调用 Dify API、文件处理）必须写在这里。

tasks.py: 定义 Celery 异步任务（调用 services 中的函数）。

views.py: 仅作为 API 入口，负责接收请求、校验权限、调用服务并返回 Response。

【前端 React 模块化要求】
src/api/: 封装所有的 Axios 请求。

src/components/: 存放通用的 UI 组件（如 SearchBar, FileCard）。

src/hooks/: 封装业务逻辑状态（如 useStandardSearch）。

src/pages/: 具体的页面（如搜索页、详情页、AI 分析页）。
